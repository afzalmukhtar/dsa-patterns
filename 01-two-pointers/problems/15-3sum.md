# 15. 3Sum

> **Difficulty:** Medium | **Link:** [LeetCode](https://leetcode.com/problems/3sum/)

---

## 📝 Problem Statement

Given an integer array `nums`, return all the triplets `[nums[i], nums[j], nums[k]]` such that `i != j`, `i != k`, and `j != k`, and `nums[i] + nums[j] + nums[k] == 0`.

Notice that the solution set must **not contain duplicate triplets**.

---

## 💡 Examples

**Example 1:**
- **Input:** `nums = [-1,0,1,2,-1,-4]`
- **Output:** `[[-1,-1,2],[-1,0,1]]`
- **Explanation:** 
  - `nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0`.
  - `nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0`.
  - `nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0`.
  - The distinct triplets are `[-1,0,1]` and `[-1,-1,2]`.

**Example 2:**
- **Input:** `nums = [0,1,1]`
- **Output:** `[]`

**Example 3:**
- **Input:** `nums = [0,0,0]`
- **Output:** `[[0,0,0]]`

---

## ⚙️ Constraints

- $3 \le nums.length \le 3000$
- $-10^5 \le nums[i] \le 10^5$

---

## 🧠 Thought Process

1.  **Identify the Pattern:** This is an extension of the **Two Sum II (Sorted Array)** problem. Instead of finding two numbers, we need three.
2.  **Sort the Array:** Sorting helps us easily skip duplicates and use the two-pointer approach effectively.
3.  **Fix One, Solve for Two:** 
    -   Iterate through the array, fixing one number `nums[i]`.
    -   Now the problem reduces to finding two numbers `nums[left]` and `nums[right]` such that `nums[left] + nums[right] = -nums[i]`.
4.  **Handle Duplicates:**
    -   **Outer loop:** Skip `nums[i]` if it's the same as the previous element (`nums[i] == nums[i-1]`).
    -   **Inner loop:** After finding a valid triplet, move both pointers and skip any identical values for `nums[left]` and `nums[right]` to ensure unique triplets.
5.  **Complexity:** 
    -   Sorting takes $O(n \log n)$.
    -   Nested loops (one for fixed element, one for two-pointers) take $O(n^2)$.
    -   Total: $O(n^2)$.

---

## 📊 Complexity Analysis

- **Time Complexity:** $O(n^2)$ — We iterate through the array once and for each element, we perform a $O(n)$ two-pointer scan.
- **Space Complexity:** $O(1)$ or $O(n)$ — Depends on the sorting implementation's space overhead. We use constant extra space for our pointers.
