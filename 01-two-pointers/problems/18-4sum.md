# 18. 4Sum

> **Difficulty:** Medium | **Link:** [LeetCode](https://leetcode.com/problems/4sum/)

---

## 📝 Problem Statement

Given an array `nums` of `n` integers, return an array of all the unique quadruplets `[nums[a], nums[b], nums[c], nums[d]]` such that:

- `0 <= a, b, c, d < n`
- `a, b, c, and d` are distinct.
- `nums[a] + nums[b] + nums[c] + nums[d] == target`

You may return the answer in any order.

---

## 💡 Examples

**Example 1:**
- **Input:** `nums = [1,0,-1,0,-2,2], target = 0`
- **Output:** `[[-2,-1,1,2],[-2,0,0,2],[-1,0,0,1]]`

**Example 2:**
- **Input:** `nums = [2,2,2,2,2], target = 8`
- **Output:** `[[2,2,2,2]]`

---

## ⚙️ Constraints

- $1 \le nums.length \le 200$
- $-10^9 \le nums[i] \le 10^9$
- $-10^9 \le target \le 10^9$

---

## 🧠 Thought Process

1.  **Identify the Pattern:** This is a direct extension of **3Sum**. Instead of fixing one number and solving for two, we fix two numbers and solve for the remaining two using the two-pointer technique.
2.  **Sort the Array:** Sorting is essential to handle duplicates and apply the two-pointer approach effectively.
3.  **Fix Two, Solve for Two:**
    -   Outer loop (fixed `a`): Iterate from `0` to `n-4`.
    -   Inner loop (fixed `b`): Iterate from `a+1` to `n-3`.
    -   Two-pointer scan (indices `c` and `d`): Use `left = b+1` and `right = n-1` to find pairs that sum to `target - (nums[a] + nums[b])`.
4.  **Handle Duplicates:**
    -   **Outer loops:** Use `if` statements with `continue` to skip duplicate values for the fixed pointers (`a` and `b`).
    -   **Inner two-pointer loop:** After finding a valid quadruplet, increment `left` and decrement `right`, then skip any identical values to ensure unique results.
5.  **Complexity:**
    -   Sorting takes $O(n \log n)$.
    -   Triple nested structure (two loops + one scan) takes $O(n^3)$.
    -   Total: $O(n^3)$.

---

## 📊 Complexity Analysis

- **Time Complexity:** $O(n^3)$ — Two nested loops for the first two elements, and a linear two-pointer scan for the remaining two.
- **Space Complexity:** $O(n)$ or $O(1)$ — Depends on the sorting implementation's space overhead. We use constant extra space for our pointers.
